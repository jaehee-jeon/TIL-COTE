## 문제 1 - 가까운 1 찾기
### ❌ TRY
```python
# [문제 요약]
# idx위치에 있거나 그 이후에 있는 1을 찾아야함
# [가정] 
# 1) 일단 idx 값에 있는 배열을 꺼내기 : 만약 idx 값이 1이면 그대로 arr[idx]==1
# 2) 아니라면 그 뒤에서 1값 찾기 : for문으로 1 찾기 순회
# 3) 뒤에서 찾았는데도 없으면 -1 출력 : for문 순회가 끝났는데도 1이 없으면 -1 출력

def solution(arr, idx):
    # idx가 i 값보다 작거나 idx 이후로 끝까지 1이 안나오는 경우 
    # 3) 뒤에서 찾았는데도 없으면 -1 출력 : for문 순회가 끝났는데도 1이 없으면 -1 출력
    answer = -1
    
    # arr 순회인데 idx에서 시작 
    # 1) 일단 idx 값에 있는 배열을 꺼내기 : 만약 idx 값이 1이면 그대로 arr[idx]==1
    for i in range(idx, len(arr)) :
        # arr이 1이면 if문
        if arr[i]==1 :
            # idx가 i랑 값이 크거나 같으면 해당 값 출력
            # 2) 아니라면 그 뒤에서 1값 찾기 : for문으로 1 찾기 순회
            if idx <=i :
                answer = i
   
    return answer
```
- 오류 :
    - for문이 끝까지 돌아가서 만약 뒤에 1이 있다면 answer 갱신됨
        - for i in range(idx, len(arr)) :
        - 중간에 중단해줘야함 : break를 쓰거나 return 활용
    - 조건문 중복
        - range(idx, len(arr))제한 루프 내부에서 if idx <= i나 elif idx < i를 다시 검사할 필요없음

### ➕ ANSWER (상기 2번으로 할 시에)
```python
def solution(arr, idx):
    # range의 시작점을 idx로 설정하여 불필요한 앞부분 탐색 방지
    for i in range(idx, len(arr)):
        if arr[i] == 1:
            return i  # 가장 처음 발견한 1이 '가장 작은 인덱스'이므로 즉시 종료
            
    return -1
```
- 배운 점:
    - return문 실행시 즉각 종료

### ✅ ANSWER 
```python
def solution(arr, idx):
    try:
        # idx 위치부터 시작하여 처음으로 등장하는 1의 인덱스를 반환
        return arr.index(1, idx)
    except ValueError:
        # 찾고자 하는 값이 없을 경우 ValueError가 발생하므로 -1 반환
        return -1
```
- 배운 점:
    - 예외 처리
        - try-except
            - 일단 실행해보고(try) 만약 에러나면(except)
        - ValueError
            - 값을 못 찾음
    - range vs index
       - range(idx, len(arr))
        - idx부터 len(arr)까지 1을 찾음
       - arr.index(1, idx)
        - 1이란 값이 어디있는지 idx 위치부터 순식간에 찾음
        - * index 사용시 값 못찾으면 에러 내뱉어서 짝궁

## 문제 2 - 할 일 목록
### ❌ 
```python
def solution(todo_list, finished):
    answer = []
    
    answer = [todo for i todo in finished todo_list if i == false]
    
    return answer
```
- 오류 :
    - 동시 순회 방법이 틀림

### ✅ ANSWER
```python
def solution(todo_list, finished):
    answer = []

    answer = [todo for i, todo in zip(finished, todo_list) if not i]
    
    return answer
```
- 배운 점:
    - 동시 순회*
        - for a, b in zip(arr, brr)
        - *zip : 두 리스트의 원소를 짝지어(튜플) 하나씩 꺼내줌
    - if i == false 보다 if not i가 더 파이썬스러움.

## 문제 3 - 홀수 vs 짝수

def solution(num_list):

- 오류 :

    - range는 인덱스일뿐..
        - a += range(0,len(num_list),2) 
        - 결과 : 0,2,4
    - += VS sum
        - += 숫자를 더할 때
        - sum 리스트를 합칠 때

### ❌ TRY 2
```python
def solution(num_list):
    answer = 0
    
    a += [i for i in num_list if i%2==0]
    b += [i for i in num_list if i%2!=0]
    if a > b :
        answer = a
    elif a < b :
        answer = b
    else :
        answer = a
    
    return answer
```
- 오류 :
    - [i for i in num_list if i%2!=0]
        - 위치가 홀수 인 경우를 찾아야하는데 상기처럼 하면 값이 홀수 인 경우를 찾게 됨

### ✅ ANSWER
```python
def solution(num_list):
    odd_sum = sum(num_list[0::2])
    even_sum = sum(num_list[1::2])
    
    return max(odd_sum, even_sum)
```
- 배운 점:
    - 슬라이싱
    - sum과 max
