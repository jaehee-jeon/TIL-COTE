## 문제 1 - 배열의 길이에 따라 다른 연산하기(재도전 🔄)
### ❌ TRY
```python
def solution(arr, n):
    answer = []
    
    #arr의 길이가 짝수라면  
    if(len(arr)%2==0):
        #arr의 모든 홀수 인덱스 위치에 n을 더한 
        answer = (i+n for i in range(1,len(arr)+1,2))
    else :
        answer = range(2, len(arr)+2,2)+n
    
    return answer
```
- 오류 :
    - TypeError: unsupported operand type(s) for +: 'range' and 'int'
        - range 안에 + 연산자 불가
    - () : 제너레이터 객체 생성

### ✅ ANSWER
```python
def solution(arr, n):
    if len(arr) % 2 == 1:
        # 홀수 길이: 0, 2, 4... 인덱스에 더함
        for i in range(0, len(arr), 2):
            arr[i] += n
    else:
        # 짝수 길이: 1, 3, 5... 인덱스에 더함
        for i in range(1, len(arr), 2):
            arr[i] += n
    return arr
```
- 배운 점:
    - range 안에 + 연산자 불가
    - () 제너레이터 VS [] 리스트 분간
    - 리스트는 0부터 시작하기 때문에 짝수일 경우 0부터

## 문제 2 - rny_string (재도전 🔄)
### ❌ TRY
```python
def solution(rny_string):
    answer = ''
    
    answer = join(i for i in rny_string if i=='m' i='rn')
    
    return answer
```
- 오류 :
    - SyntaxError: invalid syntax

### ✅ ANSWER
```python
def solution(rny_string):
    # 각 문자(char)를 순회하며 'm'이면 'rn'을, 아니면 그대로 유지
    return "".join(["rn" if char == "m" else char for char in rny_string])
```
- 배운 점:
    - () 제너레이터 VS [] 리스트 분간
    - in 주체에 대해 잘 생각해보기...
    - 리스트 컴프리헨션 if와 if-else
    - join() VS append()
        - join : 리스트(또는 제너레이터) 안에 있는 문자열들을 하나로 합쳐주는 역할
        - append : 리스트(List) 객체의 끝에 새로운 요소를 그대로 추가

## 문제 3 - 배열의 원소만큼 추가하기(재도전 🔄)
### ❌ TRY
```python
def solution(arr):
    answer = []
    
    for i in arr :
        # ✔️어떤 순서로 넣어야하는거지..? 
        # - 배열 길이부터 늘려줘야하나..? 배열 숫자부터 넣어줘야하나? 동시에..?
    return answer
```
- 오류 :
    - 어떤 순서로 넣어야하나? 
        - 파이썬 리스트는 동적 배열*
        - append나 리스트 연산을 통해 실시간으로 크기 확장 가능

### ✅ ANSWER 1 (extend)
```python
def solution(arr):
    answer = []
    for a in arr:
        # a라는 숫자를 a번 담은 리스트를 생성하여 뒤에 붙임
        answer.extend([a] * a)
    return answer
```
- 배운 점:
    - append가 원소 하나를 넣으면,
    - extend는 리스트 자체를 이어 붙임

### ✅ ANSWER 2 (리스트 컴프리헨션)
```python
def solution(arr):
    # 이중 for문을 한 줄로 처리하는 방식입니다.
    return [a for a in arr for _ in range(a)]
```
- 배운 점:
    - 리스트 컴프리헨션 중첩 가능
    - [결과값 | 외부 루프 | 내부 루프]
    - ex) [a for a in arr for _ in range(a)]
